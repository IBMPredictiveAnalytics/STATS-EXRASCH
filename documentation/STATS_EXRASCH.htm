Extended Rasch Models

This procedure estimates a standard Rasch model and five extended models.

Item Variables Specify a set of identically coded numeric item variables to analyze. The values must be integers starting with 0 and incrementing by 1. Variables can be dichotomous or polytomous.

• For repeated measures data, wide format, in which the repetitions occur as additional sets of variables, is required.

• Polytomous data must have more than two values.

Model Type Select the model type to estimate. Note that the first two types apply to dichotomous data while the others are for polytomous data.

• The linear extensions models are special cases of their base models. The linear logistic test model, for example, models the the item parameters as a weighted linear combination of underlying basic parameters and a normalization term.

Group Indicator A grouping variable can be specified to distinguish groups. Its values must be integers starting from one. You can use AUTORECODE (Transform>Automatic Recode) to put a variable into this form.

• You must have more than one measurement point to use a group variable.

• Person estimates and related statistics are not available if groups are used.

• Groups cannot be used with model types standard Rasch, linear rating scale, or partial credit.

Options

Parameter Normalization Normalization is required in order to make the model parmaeters identified. By default, the sum of the beta (item difficulty) parameters is set to zero. Alternatively, the first parameter can be set to zero.

Calculate standard errors Check this box to compute the item standard errors.

Viewer and Dataset Output

For models that do not include groups, person parameters can be computed. When this is done, a number of person-related statistics are also displayed in the Viewer.

Save person estimates Check this box and enter a dataset name for the output. The dataset will contain a person identifier assigned by the procedure, the estimated person parameter, and, if standard error estimation was specified, the standard error. Person parameters are estimated for each missing value subgroup if there are missing data. Spline interpolation is used if the parameter cannot be directly estimated. This is indicated by the Interpolated variable in the dataset.

• The dataset name must not already be in use.

• Any interpolated cases will appear out of order at the end of the dataset.

Plots Check the appropriate boxes to display person and item information plots and the item-person plot.

• Some plots are only available for RM, RSM, and PCM models. They will be ignored if specified for other models.

Item Characteristic Curve Legend Postion Choose where the legend appears in the plot. The legend is always within the plot border, so it may be necessary to move it in order not to obscure the curves. Choose None to omit the legend.

Wald Test Check the display box and choose the median or mean as the case splitting criterion for a Wald test for item elimination.

Threshold parameters Check this box to display threshold parameters. These are not available for dichotomous data. The table includes the parameters for each item level and a location parameter for the item. A second table containing the threshold standard errors and 95% confidence intervals is also produced. Threshold parameter calculation requires that standard errors be computed.

Eta Check this box to display the eta parameters from which the item parameters (beta) are derived.

Additional Features

This dialog generates syntax for the STATS EXRASCH extension command. To display the syntax help, run the following syntax:

STATS EXRASCH /HELP.
